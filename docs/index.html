Getting Started

Download the latest release of SAPIJS form https://github.com/shaktiphartiyal/sapijs/releases/latest

Browse the repository on https://github.com/shaktiphartiyal/sapijs

Simply include the file sapi.min.js to your web page like so: 

                                    <script src="sapi.min.js" type="text/javascript"></script>

Usage

GET Request 

                                    sapi.get('https://jsonplaceholder.typicode.com/posts/1').then(response => {
                                        console.log(response.data);
                                    }, error => {
                                        console.error(error);
                                    });

POST Request 

                                    sapi.post('https://httpbin.org/post').then(response => {
                                        console.log(response);
                                    }, error => {
                                        console.error(error);
                                    });

Methods

GET

Issues a GET request to the API passed in the URL parameter.

Example: 

                                    sapi.get(url, data, headers);

Data parameter should generally be passed as null in case of a GET request and the data should be appended as a query string to the URL.

POST

Issues a POST request to the API passed in the URL parameter.

Example: 

                                    sapi.post(url, data, headers);

PUT

Issues a PUT request to the API passed in the URL parameter.

Example: 

                                    sapi.put(url, data, headers);

PATCH

Issues a PATCH request to the API passed in the URL parameter.

Example: 

                                    sapi.patch(url, data, headers);

DELETE

Issues a DELETE request to the API passed in the URL parameter.

Example: 

                                    sapi.delete(url, data, headers);

HIT

This method can be used if you want to use a custom request method allowed by your server.

Example: 

                                    sapi.hit(type, url, data, headers);

Options / Parameters

All methods except hit accept three parameters (the first one - url being required) and the other two optional.

Parameters and their formats:

  • url - The API url that you need to access. E.g. http://exampleapi.com
  • data - The data that you need to send to the API in JSON format. E.g. {name:"John Snow", gender: "M"}
  • headers - The headers that you need to send to the api in JSON format. E.g. {"Authorization": "Bearer 932yrhf9egf9734fdsjkf92la"}
The type parameter is the first parameter for the hit method it is a string that could be a custom request type. E.g "HEAD"
